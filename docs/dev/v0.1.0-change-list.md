# TraceMux v0.1.0 change list

This document captures the release-readiness change set for TraceMux v0.1.0.
It is intentionally more detailed than `CHANGELOG.md`: the changelog is the
release-facing summary, while this file records the audit findings, user-facing
improvements, known limitations, and follow-up work that should guide the v0.1
stabilization pass.

Date: 2026-05-31

## Release intent

TraceMux v0.1.0 is the first executable vertical slice of the product mission:
one unified terminal for local and networked logs. The release should be judged
by whether a new user can install the tool, start a first source, observe live
data, send a command when supported, persist the session, export it, and recover
from common errors without reading implementation code.

The frozen v0.1 surfaces remain unchanged unless an ADR and compatibility
fixtures explicitly say otherwise:

- WSS wire protocol: `tracemux.v1`, MessagePack envelope.
- Session-dir log format: `raw.bin`, `index.jsonl`, decoded outputs, and
  clock metadata.
- CLI output schemas under `docs/protocols/cli-output/v1/`.
- Core trait surfaces: Source, Sink, Framer, Decoder, LogSink, Importer,
  Exporter, TimeseriesSink, and TimeSource.

## Implemented baseline

- Four-layer pipeline: `Source -> Framer -> Decoder -> LogSink/UI`.
- Server-owned architecture: browser, CLI, and Tauri shell talk to the server
  over WSS; the UI renders state and does not persist raw logs.
- Dual timestamp model on records, including origin and ingest time plus clock
  metadata.
- Session-dir persistence with WAL/group-commit/rotation primitives.
- Bearer-token auth, loopback `--no-auth` guard, rustls/TOFU primitives, and
  OS-keyring secret indirection.
- SolidJS/Dockview UI with Sources, Terminal, Tile Grid, Metrics, Packet
  Capture, Settings, notification history, and browser-local preferences.
- CLI surface for serve/connect/send/watch/detect/log/profile/replay/extcap,
  import/export, `ai-verify`, and JSON schema generation.
- Packet capture MVP with driver-free fake backend coverage, optional native
  backend behind `pcap-capture`, pcapng export, and bounded packet UI.

## Implemented in this stabilization pass

- Added the maintainer-facing v0.1.0 change list and release-facing changelog
  entry.
- Made Terminal send controls connection-aware, with visible connection state
  and actionable send failure messaging.
- Added source-start pending feedback, destructive-action confirmations, preset
  name validation, first-channel terminal action labels, and alias/note length
  counters.
- Clarified browser-local annotation fallback copy for source notes and log type
  notes.
- Added focus-visible and disabled-state styling for core form controls.
- Added Settings validation for invalid time zones, browser-local reset actions
  for display/source-start defaults, and confirmation before deleting
  classification rules.
- Added Packet Capture bounded-buffer visibility, including UI capacity and
  dropped-packet counters.
- Expanded browser E2E coverage for connection failures, settings validation,
  source details, annotations, and Packet Capture packet display.
- Made multi-channel terminal opening explicit: source rows with more than one
  channel expose a channel selector, the open/new-terminal buttons label the
  channel they target, and the confirmation toast reports the opened channel.
- Terminal send controls (text input and Send button) now disable whenever the
  WSS connection is not open, not only when no source is selected, and surface
  the reason via the control title; covered by a browser E2E regression test.
- Tile Grid now shows local connection state: a connection badge mirrors the
  Terminal panel, and while disconnected the grid is visibly dimmed and flagged
  stale so the last-received data is not mistaken for live data.
- Destructive actions (source removal and browser-local source-profile deletion)
  require an explicit confirmation prompt before they run; a browser E2E test
  asserts that dismissing the prompt sends no ctl and accepting it does.
- The shared export filename pattern now enforces its length limit with a
  `maxlength` attribute and a live counter that highlights when the limit is
  reached; the alias and note counters gained the same at-limit highlight.
- When server annotation sync fails, the source-note detail now shows a
  persistent inline notice that the note is kept in this browser only and will
  be retried on reconnect, instead of relying solely on a transient toast.
- Manual source start and serial bulk-open now surface a persistent
  "waiting for server acknowledgement" pending list that clears as the server
  registers the new sources, instead of only a one-shot toast.
- The source detail panel now shows the server-side decoded/persisted encoding
  as a distinct value and labels the effective display encoding as inherited,
  a source override, or a channel override.
- Changing the Terminal display encoding now asks for confirmation when the
  channel buffer is large enough that the full re-decode and redraw is
  noticeable, and restores the previous encoding if the user cancels.
- The Terminal tag/source text filters now debounce the full buffer redraw, so
  typing in a filter no longer repaints the terminal on every keystroke;
  dropdown-driven log-type changes still apply immediately.
- The 16-tile grid now shows the live connection state and a stale-data notice
  when disconnected, and adds a "Pause rendering" escape hatch that freezes the
  tiles' xterm draw path under high throughput (data keeps flowing on the
  server; resuming redraws from the buffer).
- The classification-rule form in Settings now validates the pattern inline:
  an invalid regular expression is flagged with a message under the field (and
  the field is marked `aria-invalid`) while the "Add rule" button stays
  disabled until the pattern compiles.
- Log type note editing is disabled (with an `aria-busy` loading hint) while the
  server annotation sync is loading or in-flight, and a failed load now exposes
  a "Retry sync" button so the field is never silently stuck.
- The Settings classification-rule section now explains its scope: rules are
  stored in the browser and applied locally to the live view, do not alter logs
  already persisted on the server, and only reach the server when "Send
  classification rules when starting sources" is enabled.
- Numeric display settings (scrollback, max records, tile min width/height) now
  show an inline notice and mark the field `aria-invalid` when an entered value
  is clamped into its safe range, instead of silently rewriting the value.
- The log type note editor now enforces `maxlength` and shows a live character
  counter that highlights (and announces) when the length limit is reached,
  matching the existing source-note and export-filename-pattern counters.

### Packet capture

- Structured pcap source-open IDs now distinguish backend unavailable,
  permission denied, invalid BPF filter, interface unavailable, and generic
  source-open failures. WSS errors preserve the pcap cause chain and the
  live-setup guide documents the manual failure matrix.
- Explain `publish=stats-only`, `publish=sampled`, and `publish=full` in the UI;
  `stats-only` must not look like capture failure.
  - _Implemented: the Packet Capture panel now carries an always-on
    publish-mode legend and a `stats-only` empty-state hint clarifying the list
    is empty by design (not a failure)._
- Show snaplen truncation when `capturedLen < originalLen`.
  - _Deferred (wire-protocol blocked): the live UI datagram envelope
    (`encode_data_envelope_with_kind(..., "datagram")` in
    `crates/server/src/pcap_runner.rs`) carries only the captured bytes, not
    `original_len`. `original_len`/`captured_len` exist only in the on-disk
    `frames.jsonl` record and export path, not in the frozen v0.1 wire
    `DataPayload`. Surfacing truncation in the live UI requires extending the
    wire protocol (a CRITICAL frozen-v0.1 path needing an ADR + human review),
    so it is tracked as a known limitation rather than implemented in this pass.
    The frontend already renders `capturedLen/originalLen` and will show
    truncation automatically once the wire field is added. The wire extension is
    specified in
    [ADR-0003](../adr/0003-live-snaplen-truncation-signalling.md) (proposed)._
- Display detected interface flags and addresses so users can avoid down or
  irrelevant devices.
  - _Implemented: the Sources panel pcap interface selector now shows the
    selected interface's description, flags, and addresses, flags interfaces
    that look down/not-running, and notes when no addresses were detected._
- Warn when the UI packet ring has dropped packets and suggest reducing volume
  with BPF, snaplen, or publish mode.
  - _Implemented: the Packet Capture panel shows a warning banner when the
    bounded UI ring drops packets, recommending a BPF filter, smaller snaplen,
    or a lower publish mode._

## Known limitations for v0.1.0

- Several source kinds are registered as stubs and return source-open errors
  until implemented in later releases.
- Live packet capture requires the optional `pcap-capture` feature and platform
  capture dependencies such as Npcap/libpcap.
- Packet capture is an MVP: no TCP stream reassembly, no Wireshark-style full
  dissection, and limited high-rate production characterization.
- Config migration is a v0.1 stub because this is the first stable config
  surface.
- UI E2E now covers high-volume ingest (1000 live sources), reconnect
  recovery (source-list and subscription replay), and basic keyboard-only
  operation (modal dialog Escape/focus return); deeper coverage of sustained
  high-throughput streaming and full keyboard navigation across every panel is
  still pending.
- Web settings and presets are browser-local except for server annotation sync;
  cross-browser settings sync is deferred.

## Deferred roadmap

- Implement remaining source stubs such as SSH, Telnet, VISA, RTT, CAN, ETW,
  journald, and remote proxy sources.
- Add a timeline/correlation panel for multi-source timestamp analysis.
- Add server-backed user settings sync and preset import/export.
- Add an export size/row preview before download. (Bulk-export cancellation is
  already implemented via an AbortController-backed cancel control.)
- Add richer packet protocol decoding and link-type-aware rendering.

## Verification expectations

- Documentation-only changes: run `pwsh scripts/check-encoding.ps1` on Windows.
- Requirement or test-reference changes: run `just rtm` and commit the generated
  `docs/rtm.md` change.
- Web UI changes: run `pnpm --filter ./web test`; run Playwright E2E when panel
  flows, keyboard behavior, or connection handling changes.
- Full release readiness: run `just ai-verify`, then `just release-gate` before
  tagging.
