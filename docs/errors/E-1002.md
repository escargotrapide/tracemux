# E-1002 backpressure deadline exceeded

## Meaning

The system could not make forward progress before a backpressure deadline was
reached.

## Common causes

- A downstream sink or subscriber is too slow.
- Disk or network I/O is saturated.
- The logger pipeline is configured with bounds that are too small for the
  current burst.

## Impact

The affected operation may be delayed or rejected. The lossless logger path is
expected to apply pressure rather than silently drop records.

## Operator action

- Check disk throughput, CPU load, and subscriber count.
- Reduce source rate or disconnect slow UI clients.
- Increase relevant queue or buffer settings only after confirming memory headroom.
- Capture metrics before and after the event for capacity planning.

## Developer notes

Keep logger-path and UI-path behavior distinct: logger backpressure is lossless;
UI fanout may drop or lag by design.
