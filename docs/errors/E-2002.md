# E-2002 wire DoS limit exceeded

## Meaning

A wire frame exceeded the configured size or resource limit.

## Common causes

- Client attempted to send a payload larger than the protocol maximum.
- A source or UI feature batched too much data into a single frame.
- A malformed or abusive peer ignored documented limits.

## Impact

The frame is rejected and the connection may be closed to protect the server.

## Operator action

- Check the client version and feature that generated the frame.
- Reduce batch size, export scope, or control payload size.
- Treat repeated occurrences from unknown peers as suspicious traffic.
- Review reverse proxy or gateway limits if present.

## Developer notes

Limit enforcement is security-sensitive. Any limit change must update protocol
docs and wire compatibility coverage.
