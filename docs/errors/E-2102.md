# E-2102 TLS handshake failed

## Meaning

TLS setup or handshake failed.

## Common causes

- Certificate or private-key files are missing, unreadable, or malformed.
- Certificate generation failed.
- Client and server cannot agree on TLS parameters.
- The connection is not using the expected secure endpoint.

## Impact

The secure connection is not established. WSS clients cannot use the server until
TLS setup succeeds or a development-only non-TLS endpoint is used.

## Operator action

- Check certificate and key paths, permissions, and PEM formatting.
- Regenerate development certificates if they are stale or corrupt.
- Confirm the client connects to the intended `wss://` endpoint.
- Inspect server logs for the exact TLS setup stage.

## Developer notes

Keep TLS errors mapped to `E-2102` unless fingerprint pinning specifically
requires `E-2103`.
