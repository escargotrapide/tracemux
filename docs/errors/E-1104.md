# E-1104 packet capture permission denied

## Meaning

A `pcap://` source reached the packet-capture backend, but the selected account
or host policy did not allow opening the interface.

## Common causes

- The current user is not allowed to capture packets on the selected adapter.
- Linux capabilities or packet-capture group membership are missing.
- macOS or Windows policy blocks capture for the process.
- Promiscuous mode was requested but is not permitted on the adapter.

## Impact

The packet-capture source is not registered and no packets are ingested.

## Operator action

- Run tracemux from an account approved for packet capture.
- On Linux, grant capture privileges through root, capabilities, or the
  distribution-approved capture group.
- On Windows, confirm Npcap is installed and the account has capture rights.
- Retry with `promisc=0` if promiscuous mode is not required.

## Developer notes

This code is selected from backend errors that mention permission, access,
privileges, or operation-not-permitted style failures.
