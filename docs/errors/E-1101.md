# E-1101 source open failed

## Meaning

Opening or initializing a source failed.

## Common causes

- The device, file, interface, host, or port does not exist.
- Permissions are missing, such as serial-device access or packet-capture rights.
- A required feature or host dependency is unavailable.
- Source options are invalid for the selected transport.
- The source failed in a way that does not match a more specific source-layer
	code such as `E-1103` through `E-1106` for packet capture.

## Impact

The source is not registered as running and no new records are ingested from it.

## Operator action

- Confirm the path, port, interface, host, and credentials.
- Check platform dependencies such as serial drivers or Npcap/libpcap.
- Retry after fixing permissions or correcting the source specification.
- Use discovery commands or the UI discovery panel to refresh available sources.
- For packet capture, prefer the more specific pcap runbook when the error ID
	is `E-1103`, `E-1104`, `E-1105`, or `E-1106`.

## Developer notes

Prefer returning `E-1101` before partially registering a source. Include enough
context in the message for operators to distinguish missing dependencies from
invalid user input.
