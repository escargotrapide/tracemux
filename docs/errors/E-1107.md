# E-1107 pseudo-terminal unavailable

## Meaning

A `pty://` source (or a `process://...?pty=1` alias) was requested, but the
pseudo-console could not be allocated or the child could not be spawned into it.

## Common causes

- The binary was built without the `pty` feature.
- The host could not allocate a pseudo-console (ConPTY on Windows, openpty on
  Unix), e.g. resource exhaustion or a sandbox that forbids it.
- The program named by `argv[0]` does not exist or is not executable.

## Impact

The interactive terminal source is not registered and no bytes are ingested.
A partially opened source is never left behind.

## Operator action

- Confirm the binary was built with `--features pty` when an interactive
  terminal is required.
- Verify `argv[0]` is a valid program path or is resolvable on `PATH`.
- Check server logs around the same timestamp for the underlying OS error.
- For non-interactive command output, use the pipe-based `process://` source
  instead, which does not require a pseudo-console.

## Developer notes

Default CI and normal development builds keep the `pty` feature disabled, so
`PtySource::spawn_duplex`/`open` return this code immediately. Use this code for
PTY allocation and child-spawn failures; empty `argv` uses `E-1101`
(`E1101SourceOpen`), and read failures after open use `E-1102`
(`E1102SourceClosed`). See ADR-0004 and `FR-SRC-PTY`.
