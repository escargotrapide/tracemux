# E-1401 WAL fsync failed

## Meaning

The write-ahead log could not fsync data to durable storage.

## Common causes

- Disk is full or unavailable.
- Filesystem or storage device returned an I/O error.
- Permission or antivirus software blocked the write or flush operation.

## Impact

Recent records may not be durable. The session should be treated as at risk
until storage health is confirmed.

## Operator action

- Stop or pause high-rate sources if storage is unhealthy.
- Check free space, filesystem health, and operating-system logs.
- Move the session root to reliable storage if failures persist.
- Preserve the affected session directory for recovery analysis.

## Developer notes

Never downgrade fsync failures to warnings on the logger path. Keep recovery and
torn-tail behavior covered by tests.
