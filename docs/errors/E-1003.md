# E-1003 framer overflow

## Meaning

A framer received more bytes than its configured frame limit before it could
emit a complete frame.

## Common causes

- Missing delimiter in line-oriented input.
- Length-prefixed input declared a frame larger than the allowed maximum.
- The selected framer does not match the source protocol.

## Impact

The oversized frame is rejected. The channel may continue if the framer can
resynchronize; otherwise the source may need to be restarted.

## Operator action

- Verify the selected framer matches the device or log format.
- Check whether the source emits delimiters or length prefixes as expected.
- Increase the frame limit only if the source is trusted and the larger frame
  size is expected.
- Capture a small raw sample for protocol diagnosis.

## Developer notes

Do not bypass the limit in wire-facing or untrusted paths. Large limits should
remain explicit and test-covered.
