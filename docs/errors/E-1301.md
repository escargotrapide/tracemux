# E-1301 decoder schema mismatch

## Meaning

A decoder could not map a frame into the expected record schema.

## Common causes

- The selected decoder does not match the incoming frame format.
- Required fields are missing or have incompatible types.
- A schema version changed without a matching decoder update.

## Impact

The affected frame may be emitted as an error, skipped, or preserved only as raw
data depending on the pipeline configuration.

## Operator action

- Confirm the decoder selection for the source.
- Compare a raw frame sample with the expected schema.
- Switch to passthrough or UTF-8 text decoding if structured decoding is not
  required.
- Report recurring mismatches with a sanitized sample frame.

## Developer notes

Prefer preserving raw bytes when structured decoding fails. Keep schema changes
compatible or versioned.
