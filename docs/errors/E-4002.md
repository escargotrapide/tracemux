# E-4002 UI WebSocket send failed

## Meaning

The web UI tried to send a WebSocket frame, but the browser WebSocket rejected
the send operation.

## Impact

The requested command, subscription, or write-back frame may not have reached
the server. Server-owned logs are not modified by the failed browser send.

## User remedy

- Check that the connection state is `connected` before retrying.
- If the server was restarted, wait for reconnect and retry the action.
- For terminal write-back, confirm that the selected source supports a Sink.

## Developer notes

This is a browser-side transport failure. Server-side write routing failures
should return the appropriate `E-NNNN` in a `ctl error` frame.
