# E-2001 wire frame malformed

## Meaning

The server or client received a wire frame that did not match `wanlogger.v1`.

## Common causes

- MessagePack payload is invalid or incomplete.
- Required envelope fields are missing or have wrong types.
- The frame type is unknown for the negotiated protocol version.
- A lifecycle or control request failed validation.

## Impact

The malformed frame is rejected. The connection may remain open unless repeated
errors or policy require closing it.

## Operator action

- Confirm that client and server use the same `wanlogger.v1` protocol.
- Check browser or CLI logs for the frame type and sequence number.
- Restart stale clients after upgrading the server.
- Capture a sanitized frame sample when reporting an integration issue.

## Developer notes

Keep malformed-frame handling explicit and bounded. Do not accept unknown fields
as new semantics without protocol documentation and compatibility tests.
