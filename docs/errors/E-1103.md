# E-1103 packet capture backend unavailable

## Meaning

A `pcap://` source was requested, but this build or host cannot provide the
packet-capture backend.

## Common causes

- The binary was built without the `pcap-capture` feature.
- Npcap/libpcap support is not installed or not visible to the process.
- The runtime packet-capture library cannot be loaded by the host environment.

## Impact

The packet-capture source is not registered and no packets are ingested.

## Operator action

- Confirm the binary was built with `--features pcap-capture` when live capture
  is required.
- Install Npcap on Windows or libpcap on Linux/macOS.
- Restart the server after installing host capture dependencies.

## Developer notes

Default CI and normal development builds intentionally keep `pcap-capture`
disabled. Use this code when the pcap transport itself is unavailable, not when
a specific interface or filter is invalid.
