# E-2101 auth rejected

## Meaning

Authentication failed or the request was not allowed by the current auth policy.

## Common causes

- Missing, malformed, or incorrect bearer token.
- Stored password hash is invalid.
- `--no-auth` was requested from a non-loopback address.
- A download or bundle ticket is missing, expired, or invalid.

## Impact

The request is rejected. No source, annotation, export, or write-back operation
should be performed for the rejected request.

## Operator action

- Verify the token source and environment variables.
- For browser clients, confirm that the configured backend URL matches the token.
- Reissue expired tickets by repeating the export request.
- Keep `--no-auth` limited to loopback development use.

## Developer notes

Do not log secrets. Keep rejection messages useful without revealing token or
hash material.
