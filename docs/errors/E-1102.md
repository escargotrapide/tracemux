# E-1102 source closed unexpectedly

## Meaning

A source or write-back path closed while the system still expected it to be
available.

## Common causes

- Remote TCP peer disconnected.
- Serial device was unplugged or reset.
- A child process, pipe, or file-follow source ended.
- A sink-capable source refused or lost write-back capability.

## Impact

Ingest or write-back stops for the affected channel. Previously persisted data
remains available.

## Operator action

- Check the physical connection or remote peer status.
- Restart the source after the device or peer is available.
- For write-back failures, verify that the selected source supports `Sink`.
- Inspect surrounding logs for an earlier `E-1101` or transport-specific error.

## Developer notes

Use `E-1102` for closures after a source was opened successfully. Use `E-1101`
for initialization failures.
