# E-1106 packet capture interface unavailable

## Meaning

A `pcap://` source named an interface or device that the packet-capture backend
could not find or activate.

## Common causes

- The interface name was typed manually and does not match the backend device
  identifier.
- The adapter was removed, renamed, disabled, or is hidden by host policy.
- A Windows Npcap device path was confused with the friendly adapter name.

## Impact

The packet-capture source is not registered and no packets are ingested.

## Operator action

- Use the UI packet-capture candidate list or `/api/detect` to refresh device
  names.
- Copy the backend identifier exactly as reported by detection.
- Confirm the adapter is enabled and visible to Npcap/libpcap.
- Retry after reconnecting or enabling the interface.

## Developer notes

This code is selected for pcap creation or activation errors that look like a
missing device. Permission and BPF failures have their own pcap source-open
codes.
