# E-1105 packet capture filter invalid

## Meaning

A `pcap://` source supplied a BPF filter that the packet-capture backend could
not compile or apply.

## Common causes

- The `filter` query parameter contains invalid BPF syntax.
- Shell or URI escaping changed spaces, quotes, or parentheses before the server
  received the filter.
- The selected backend does not support a construct used by the filter.

## Impact

The packet-capture source is not registered and no packets are ingested.

## Operator action

- Remove the filter and confirm the interface opens.
- Re-add the filter in small pieces, for example `tcp port 502`.
- URL-encode spaces when using URI specs, such as `filter=tcp%20port%20502`.
- Prefer `publish=stats-only` plus a narrow filter for high-rate captures.

## Developer notes

Use this code only for failures while applying the BPF filter. Interface open
and permission failures should use the more specific pcap source-open codes.
