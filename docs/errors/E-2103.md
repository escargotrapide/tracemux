# E-2103 TOFU fingerprint mismatch

## Meaning

Trust-on-first-use fingerprint validation detected a different certificate or
key fingerprint than the one pinned for the host.

## Common causes

- The server certificate was rotated without updating the pin.
- The client connected to the wrong host or port.
- A proxy or middlebox presented a different certificate.
- There is a potential man-in-the-middle condition.

## Impact

The connection is rejected to protect session data and credentials.

## Operator action

- Do not bypass the warning until the server identity is confirmed.
- Compare the displayed fingerprint with the expected server fingerprint through
  a trusted channel.
- Remove or update the pin only after confirming intentional certificate rotation.
- Investigate unexpected changes as a security event.

## Developer notes

TOFU storage and mismatch handling are security-sensitive. Keep tests for schema
mismatch, I/O failure, and fingerprint mismatch cases.
