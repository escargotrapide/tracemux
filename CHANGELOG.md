## [0.1.0] - 2026-05-31

### Added
- Initial workspace scaffolding (Cargo workspace, lints, justfile).
- Frozen v0.1 trait surfaces under `crates/core`.
- AGENTS.md, copilot-instructions, 10 skill manuals, ADR-0001.
- Wire-protocol, log-format, timestamp, cli-output v1 spec drafts.
- CI workflows (rust matrix, audit, deny, compat, RTM diff,
  critical-path labelling).
- Server WSS subprotocol `tracemux.v1` with MessagePack envelope,
  argon2id bearer auth + loopback gate, rustls + rcgen TLS, TOFU
  fingerprint pinning (FR-WIRE-001/002, Phase A).
- Core WAL header / group-commit / rotation, secret resolver,
  `NodeClockTable` with dual-timestamp ingest (FR-LOG-001/002,
  FR-SEC-001, Phase B+C).
- CLI `import` / `export` text round-trip (FR-IMP-001, FR-EXP-001,
  FR-CLI-001, Phase D2).
- CLI `extcap --capture` writes libpcap (DLT_USER0) records to a
  Wireshark FIFO (FR-CLI-002, Phase G).
- Packet capture MVP: pcap source model with fake tests and optional
  Npcap/libpcap backend behind `pcap-capture`, pcap source specs, interface
  discovery, pcap metrics, session-dir packet metadata, packet summary parsing,
  pcapng export, and direct pcapng streaming with rotation.
- Web UI: metrics panel, sources panel actions (start/stop/remove),
  toast notifications, terminal TX, 4×4 tile grid, Playwright e2e
  (FR-UI-007/008/009/010/011/012, Phase E1+E2-E5).
- AI verification driver (`scripts/ai-verify-summary.{ps1,sh}`)
  emitting `target/ai-verify.json` per the
  `tracemux/ai-verify/v1` schema (FR-AI-001, Phase F).
- Release-gate driver (`scripts/release-gate.{ps1,sh}`) with six
  pre-release checks (FR-AI-002, Phase H).

### Changed
- Completed the v0.1.0 release-readiness pass, tracked by the
  maintainer-facing change list under `docs/dev/v0.1.0-change-list.md`.
- Improved web UI affordances for first-use credibility: visible keyboard
  focus rings, clearer disabled controls, Terminal connection-state feedback,
  source-start pending feedback, and safer destructive source/preset actions.
- Hardened web UI accessibility and presentation: unit-aware Metrics
  formatting with a legend, `aria-busy` on long-running actions, scoped
  table headers, and a focus-trapped notification center that closes on
  Escape and returns focus to its trigger.
- Expanded CLI discoverability with `long_about` help text on every
  subcommand and clearer not-yet-implemented import-kind hints.

### Documentation
- Captured user-facing v0.1.0 follow-up work for onboarding, auth/TLS setup,
  source specs, packet capture, CLI examples, troubleshooting, and known
  limitations.

### Known limitations
- Some source kinds remain registered as v0.1 stubs and return source-open
  errors until implemented in later releases.
- Live packet capture remains optional behind `pcap-capture` and depends on
  platform capture drivers/SDKs such as Npcap or libpcap.
- Live capture does not yet signal snaplen truncation on the wire; the
  additive `orig_len` field is proposed in ADR-0003 and deferred.
- High-volume UI recovery, keyboard-only navigation, and reconnect behavior need
  broader E2E coverage before being considered fully production-characterized.
